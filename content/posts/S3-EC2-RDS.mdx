## What is Deployment?

### Environments

응용 프로그램의 수명주기에는 코드가 통과 할 여러 가지 환경이 있습니다. 각 환경에는 특정 목적 (테스트, 기능 추가, 디버깅)을 위해 다른 컴퓨터에서 코드를 실행하는 것이 포함됩니다.

프로젝트의 네 단계는 다음과 같습니다.
1. 개발
2. 통합
3. 준비
4. 생산

개발
개발자 컴퓨터에서 새로운 기능을 생성하고 응용 프로그램을 테스트합니다. 샘플 데이터를 사용하며 많은 코드 변경을위한 안전한 환경입니다.

완성
개발자 코드를 메인 브랜치로 푸시하는 단계입니다. 개발자는 풀 요청에 대한 변경 사항을 제출하고 전체 프로젝트와 일치하지 않는지 검토합니다. 코드는 충돌을 방지하고 품질을 검증하기위한 테스트를 구현해야합니다.

각색
응용 프로그램이 프로덕션 환경을 모방하는 단계입니다. 실제 테스트 환경에서 실제 데이터의 사본을 사용하여 이해 관계자에게 프로젝트 품질을 검증합니다.

생산
프로덕션은 실제 라이브 데이터를 사용하는 실제 라이브 버전의 응용 프로그램입니다. 고객의 요구에 맞게 코드가 적절하게 작동해야합니다.


## What is Cloud?

클라우드는 최종 사용자의 직접적인 관리없이 데이터 저장 및 컴퓨팅 성능에 사용되는 주문형 컴퓨터 시스템입니다. 데이터 센터에 있으며, 소유자 (Amazon, Google)가 유지 관리하여 새로운 회사의 선행 IT 인프라 비용을 최소화합니다. 가동 시간이 단축되고 관리 효율성이 향상되었으며 유지 관리가 줄어 듭니다.


### S3 (Amazon Simple Storage Service)

* 웹 서비스 인터페이스를 통한 객체 스토리지
* 인터넷 애플리케이션 용 스토리지, 백업, 복구, 재해 복구, 재해 복구, 데이터 아카이브, 데이터 레이크, 하이브리드 클라우드 스토리지와 같은 모든 유형의 오브젝트
* 정적 HTML 웹 사이트를 호스팅 할 수 있음 (React SPA)


#### Access point

S3 객체 작업 (GetObject, PutObject)을 수행하기 위해 버킷에 연결된 네트워크 엔드 포인트


#### Bucket

* 데이터가 저장되는 장소
* 관리에 사용되는 API


#### Object

S3는 간단한 키 값 저장소를 사용합니다


### EC2 (Elastic Compute Cloud)

* 두 번째 "탄력적"에 의해 지불되는 클라우드 컴퓨팅
* 컴퓨터 응용 프로그램을 실행할 가상 컴퓨터 대여
* 사용자는 필요에 따라 서버 인스턴스 (가상 컴퓨팅 환경)를 생성, 시작 및 종료 할 수 있습니다


### RDS (Amazon Relational Database Service)

* 클라우드에서 분산 관계형 데이터베이스 서비스
* 관계형 데이터베이스의 설정, 운영, 확장을 단순화
* 패치, 백업 및 복구가 자동으로 관리됩니다
* 단일 API 호출로 수행되는 스토리지 및 컴퓨팅 리소스 확장
* RDS 인스턴스에 ssh (보안 셸) 연결을 제공하지 않습니다
* CPU, 메모리, 스토리지를 개별적으로 할당


### ARN (Amazon Resource Name)

AWS에서 리소스를 식별하기위한 형식

```jsx
arn:partition:service:region:namespace:relative-id
```

* aws는 일반적인 파티션 이름입니다
* s3는 서비스입니다


#### S3 ARN 형식

```jsx
arn:aws:s3:::bucket_name
arn:aws:s3:::bucket_name/key_name
```


### Principal in a Policy

주요 요소는 사용자, 계정, 서비스 또는 자원에 대한 액세스가 허용되거나 거부 된 기타 엔티티를 지정합니다.
AWS 계정에 권한을 부여하려면 :
```jsx
"AWS":"account-ARN"
```
