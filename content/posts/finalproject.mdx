## 

4 주가 지났고 마침내 우리 팀은 최종 프로젝트가 끝났습니다. 우리의 응용 프로그램 ShareBook은 여러 변경 사항, 버그 수정 및 기능 삭감을 통해 실행되었습니다. 그러나 4 주 동안이 응용 프로그램을 작성하는 데 필요한 지식과 경험을 얻었습니다.

나는 프론트 엔드에서 익숙하지 않은 React 네이티브, redux, material UI 및 axios 스택으로 작업했습니다. 이 라이브러리가 얼마나 새로운 지에 따라 프론트 엔드 팀은 소스 코드에 뛰어 들기 전에 공부하고 연습하는 데 며칠을 보냈습니다.

우리는 React native로 시작했으며, 페이지 간 탐색을 위해 라이브러리가 필요했습니다. React Navigation을 결정하고 스택 탐색과 탭 탐색의 두 가지 탐색 유형을 연구했습니다.

스택 탐색은 스택 데이터 구조와 유사하게 작동했습니다. 각 화면은 각 화면의 상단에 배치되어 가상 스택을 생성합니다. 가상 스택은 위에서 아래로 튀어 나와 이전 화면으로 돌아갈 수 있습니다.


탭 탐색은 브라우저 탭처럼 작동했습니다. 각 탭은 페이지 이름이 다르며 응용 프로그램을 차별화 된 범주로 나눕니다. 스택과 탭을 사용하여 애플리케이션의 레이아웃을 중첩 할 수있었습니다.


해당 지역의 책에 대한 화면을 완성한 후 빌린 책은 사용자가 Axios를 사용하여 책을 빌리도록 요청하는 버튼을 만들었습니다. 사용자를 인증하기 위해 로그인 할 때 로그인 및 등록 기능을 담당 한 팀 구성원이 jwt 액세스 토큰을 AsyncStorage에 저장했습니다. Axios 요청에서 토큰에 액세스하기 위해 AsyncStorage.getItems()를 호출 한 다음 각 요청의 헤더에 추가했습니다.


책을 요청한 후 (빌리거나 반환하기 위해) redux 조치를 호출하여 조회 한 목록을 업데이트하여 상점에서 선택한 책을 제거했습니다. 그런 다음 우리 팀은 레이아웃 목적으로 Material UI의 카드 구성 요소를 사용하기로 결정했습니다. 프론트 엔드 팀과 백엔드 팀 모두에서 응용 프로그램의 색 구성표를 공동 작업했습니다.

앱을 완성하는 데 필요한 모든 기본 기능을 완료 한 후에는 채팅 기능이 목표의 다음 단계가 될 것이라고 결정했습니다. 이 기능을 구현하는 데 며칠 밖에 걸리지 않았지만이 지역의 각 책에 대한 기본 채팅 기능을 이용할 수있었습니다.

유감스럽게도 채팅 기능을 끝내지 않고 각 책 목록에 표시 할 실시간 업데이트 서비스를 구현하고 있습니다. 그러나 저는 아이디어를 완전히 이해하지 못하는 좌절스러운 순간에 대해 관대하고 유머러스 한 각 팀원과 함께 많은 것을 배우고 즐거운 시간을 보냈습니다. 우리가이 프로젝트를 완수했을 때 좋은 정신과 태도를 보여 주신 팀에게 감사 드리며 앞으로 훌륭한 개발자로 볼 것입니다.
